# v3.2 Design Spec — Bug Fixes, New Checks, DX

**Date:** 2026-04-27
**Target:** v3.2.0
**Scope:** fix known bugs, add 4 new scanner checks, add .fartrunignore

---

## Phase 1: Bug Fixes

### BUG-1: Self-scan tests take 6+ minutes

**Problem:** `test_health_self_scan.py` has 4 tests that each call `run_all_checks(PROJECT_ROOT)`. Cache bypasses dirty working tree, so each test does a full 89s scan. Total: ~6 min.

**Fix:** Shared `@pytest.fixture(scope="module")` that runs `run_all_checks(PROJECT_ROOT, use_cache=False)` once. All 4 tests consume the same report object.

**Files:** `tests/test_health_self_scan.py`

### BUG-2: test_models_hub_count FAILED

**Problem:** After refactors the number of files importing `core/models.py` changed. Assertion `>= 5` no longer holds.

**Fix:** Relax to `>= 3` or make dynamic — assert hub detection works (returns non-empty) without hardcoding exact counts.

**Files:** `tests/test_health_self_scan.py`

### BUG-3: Version hardcoded in 6 places

**Problem:** Version string lives in `pyproject.toml`, `npm/package.json`, `gui/dialogs/about.py`, `gui/pages/about.py`, `i18n/en.py`, `i18n/ua.py`. Easy to miss one during bump.

**Fix:** Single source of truth: `pyproject.toml`.
- `gui/dialogs/about.py`, `gui/pages/about.py` → `importlib.metadata.version("claude-monitor")`
- `i18n/en.py`, `i18n/ua.py` → `{version}` placeholder, substituted at render time
- `npm/package.json` → CI syncs from pyproject.toml before publish

**Files:** `gui/dialogs/about.py`, `gui/pages/about.py`, `i18n/en.py`, `i18n/ua.py`, `.github/workflows/release.yml`

### BUG-4: System python vs .venv stale health module

**Problem:** `maturin develop` installs to .venv, but system python has old copy without `ScanContext`. Tests crash with `AttributeError`.

**Fix:** Add guard in `tests/conftest.py` — if `health` module lacks `ScanContext`, skip Rust-dependent tests with message: "rebuild: cd crates/health && maturin develop --release".

**Files:** `tests/conftest.py`

## Phase 2: New Scanner Checks

### CHECK-1: console.log / print() spam detection

- **check_id:** `dead.debug_output`
- **Severity:** low
- **What:** Find leftover `console.log`, `console.warn`, `console.error` (JS/TS), `print()` (Python outside test files)
- **How:** Rust scanner via tree-sitter — match `call_expression` nodes with these names
- **Exclusions:** test files, `logging.*` calls, `print()` in CLI entry points (`if __name__`)
- **Files:** `crates/health/src/dead_code.rs` (or new `debug_output.rs`), Python orchestrator in `core/health/dead_code.py`

### CHECK-2: Hardcoded localhost/ports

- **check_id:** `tech_debt.hardcoded_url`
- **Severity:** medium
- **What:** Find `http://localhost`, `127.0.0.1`, `0.0.0.0` with port in non-test source files
- **How:** Regex in Rust tech_debt scanner
- **Exclusions:** config files (docker-compose, .env), README/docs, test files
- **Files:** `crates/health/src/tech_debt.rs`, `core/health/tech_debt.py`

### CHECK-3: os.environ["KEY"] without fallback

- **check_id:** `tech_debt.env_no_fallback`
- **Severity:** medium
- **What:** Find `os.environ["..."]` (crashes if missing) instead of `os.environ.get()` or `os.getenv()`
- **How:** Tree-sitter on Python files — match subscript on `os.environ`
- **Exclusions:** test files
- **Files:** `crates/health/src/tech_debt.rs`, `core/health/tech_debt.py`

### CHECK-4: Stale git branches

- **check_id:** `git.stale_branches`
- **Severity:** info
- **What:** Branches older than 30 days, not merged into current branch
- **How:** `git branch -a --sort=-committerdate` + `git log -1 --format=%ci` per branch
- **Exclusions:** main/master, HEAD
- **Files:** `core/health/git_survival.py`

## Phase 3: DX Improvements

### DX-1: .fartrunignore

- **Syntax:** `.gitignore`-like, plus check_id scoping
- **Format:**
  ```
  # ignore files/dirs
  vendor/
  generated/*.py

  # ignore specific check for a file
  [dead.debug_output] scripts/debug_helper.py

  # ignore check globally
  [brake.opensource_check]
  ```
- **How:** Parsed once per scan. Filter applied on Python side after all findings collected, before `run_all_checks` returns. No Rust changes needed.
- **Files:** new `core/health/ignore.py`, modify `core/health/project_map.py`
